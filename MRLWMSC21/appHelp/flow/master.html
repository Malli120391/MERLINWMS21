    1. Master Data

      The master data related features are listed below:

      1. Account
        1. Navigation: Home Master Data Account
        2. Screen: The following screen displays the access to Account Master for Inventrax User / Super Admin:
        3. Description
          1. Functionality: Functionality of Inventrax User / Super Admin in Account List screen:
            1. Create a New Account through Add+ button
            2. Have access to view all the active accounts in Account List page of Merlin application

              3. Search Account will list out all the active accounts in the dropdown and will fetch the search field record based on user selection.

          2. Functionality of Inventrax User / Super Admin in Account Form screen:
            1. New account can be created by providing Account Name, Unique Account Code, Company Name and Company Logo
          3. Warehouse Creation:
            1. After creating an Account, warehouse under that account must be created -
            2. Warehouse code is unique for an Account which must be of 3 Characters
            3. Dock Creation – A dock can be created after creating warehouse and Dock No is unique for a warehouse under an account. A dock can be created for Inbound, Outbound or Both. For every dock created, dock locations will be added in the database.
            4. Zone Creation: Default Docking Zone code will be generated after creating the Warehouse. Zones under a warehouse can be created clicking on Create Zone apart from Dock zone ‘DZ’.
            1. Business Rules
              1. ‘DZ’ Zone code cannot be updated or deleted
              2. Zone code is unique for a warehouse
              3. Zone can be deleted only if locations are not present under the Zone otherwise it cannot be deleted
              4. Locations under a Zone can be added through Add Loc button.
          4. Account User Creation: Account User will be created for the Account with Account Admin role. Email id is unique for the entire application since it is used for Login. Warehouse should be selected for the Account user.
          5. Account Admin: Account User which is created by Inventrax User / Super Admin for an Account can be Logged into application and the Company Logo given while Account creation can be seen in the top left.
          6. Functionality of Account Admin:
            1. Account Admin will have access to a specific account.
            2. Manage warehouses (including docks, zones) and users under the account.
            3. Account Admin will set the preferences across the account.
            4. Create Tenant users and warehouse ops users and assign roles to access the application
            5. Selecting Warehouse is mandate when user type is Account & warehouse Ops
            6. Selecting tenant is mandate when user type is tenant
      2. Tenant
        1. Navigation: Home Master Data Tenant
          2. Description
        2. This feature allows the user to view the list of tenants available for the respective account. An account can have multiple tenants mapped to it. It provides Search, Edit, and Update functions based on the user role.
          The mandatory fields marked in red must be entered to create a new tenant. Authorized users can create ‘Tenant’.
          3. Billing Information
        3. All the billing information for a particular tenant is given in the billing information grid and all the bills for that tenant will be generated to that address.
          4. Contract Information
        4. All the contract information for a particular tenant with respect to warehouse is given in the contract information grid and all the details like contract period, agreement files and the warehouse details for that tenant will be mapped.
        5. Functionality
          1. The tenants can be searched with tenant Name
          2. The ‘Edit’ function in tenant List allows modifying the tenant details with account field disabled (since account is unique)
          3. On creation of new tenant following sections will be visible to update the required data
            1. Billing Information: Allows adding billing information of the tenant.
            2. Contract Information: Allows adding contract information of the tenant.
      3. Supplier
        1. Navigation: Home Master Data Supplier
          2. Description
        2. This feature allows the user to view the list of suppliers available in the system and provides Search, Add new supplier, Edit, Update status, Export to Excel and Import excel functions based on the user role.
          3. Functionality
        3. System forwards this request to concerned Dept. (System sends an alert via automated email of this request). A unique supplier code (3 characters) for this supplier will be auto-generated. Once the supplier is created, system enables the use of this supplier in Item Master Request. The mandatory fields marked in red asterisk must be entered to create a new supplier request. Authorized users can create ‘Supplier’ and update the ‘Is Approved’ status / Supplier details
          1. The suppliers can be searched with Supplier Name, tenant with the tenant name and the accounts with the account name.
          2. The ‘Edit’ function in Supplier List allows modifying the supplier details with Supplier Code field disabled (since supplier code is unique).
          3. The entire supplier list that is displayed can be exported as an ‘Excel file’ for reference.
        4. On creation of new supplier following sections will be visible to update the required data
          1. Supplier Details: Allows adding basic information of the supplier.
          2. Contact Person Details: Allows adding contact person information. Supplier is the one who supplies material to the Tenant. Following are the requirements to create and manage Supplier
        5. Business Rules
          1. Create New Supplier and Manage the Supplier with the Data points provided from the existing system for a given tenant
          2. Unique Supplier Code should be maintained against each supplier
          3. System will bulk import Supplier list from and Excel/CSV format provided
          4. System will seamlessly sync Suppliers details from an Existing System if an Integration is setup. Suppliers list with filters are provided
      4. Customer
        1. Navigation: Home Master Data Customer
        2. Description
          1. This feature allows the user to view the list of customers available for the respective account. An account can have multiple customers mapped to tenant. It provides Search, Edit, Update and Export to Excel and Import Excel functions based on the user role.
          2. Customer is the one who consumes material from the Tenant.
          3. The mandatory fields marked in red must be entered to create a new customer.
          4. Authorized users can create ‘customer’.
        3. Functionality
          1. We can request an addition of new customer in the master data, based on the materials they need. User enters details of customer like Account, Tenant, Customer Name, Customer Code as shown in the following screen.
      5. Item Master
        1. Navigation: Home Master Data Item Master
        2. Description
          1. This feature allows the user to view the list of materials available in the system and provides Search, Edit, Copy, Print Label, Add Materials, Export to Excel, Import excel functions based on the user role.
        3. Functionality
          1. The Items can be searched with Material Type / Supplier / Part No.
          2. The ‘Edit’ function in ‘Item Master List’ allows to modify the item details (part no. is unique)
          3. The ‘Copy’ function in ‘Item Master List’ allows to create a ‘new item’ (change in Part Number) with source item data
          4. The entire item master list that is displayed can be exported as an ‘Excel file’ for reference
          5. Import Excel
            1. This feature allows the user to import the materials (materials will be created in bulk) from a default excel template with basic header data (part no., material type, product category, storage condition, item description, BUoM, BUoM/Qty., MUoM, MUoM/Qty.)
          6. The ‘Print’ function allows to print barcode labels with the provided Part Number and enters required fields Mfg. Date, Exp. Date, Serial No., Batch No., Str. Ref. No., Req. No., Quantity and selects Printer, Label size
          7. Description
            1. This feature allows the user to request a new item to be added into the Item Master, before using the item in any of the system transactions.
            2. Item detailed data is further segregated into groups such as Basic Material details, Supplier Details, UoM Configuration, Storage Condition details and MSP’s.
            3. Item should be configured based on Supplier and Tenant.
            4. The mandatory fields marked in red asterisk must be entered to create a new item request.
            5. Stock Keeping Unit (SKU) /Part number is a Unique Code to identify items in the Item Master
          8. Functionality

              On creation of new item following sections will be visible to update the required data

            1. Supplier Details and Tenant Details: Allows to add supplier(s) and Tenant for the material
            2. UoM Configuration
              1. UoM Conversion
              2. Allows to configure UoM(s) for the material
                1. Base UoM will be configured on selection of UoM type and it will always be 1
                2. Once B.UoM is configured, besides B.UoM, system automatically adds Min. Pick UoM in UoM type list with its UoM, Qty. Per UoM same as B.UoM
                3. UoM type lists only the pending unconfigured UoM types for this item master

              Attachments: - Tech. Datasheets, Eng. Drawings, CoC and other significant documents can be attached for a given supplier without any limitation on the document nos.

          9. Business Rules

              MSP’s
            1. “Default MSP’s will be updated based on default Material Type, MSP configuration” (Ex: For Material Type ‘RM-Raw Material’ ,default MSP’s like Mfg. Date, Serial No., Batch No. will be set)
            2. If ‘Is Required’ checkbox is enabled then it must be a mandatory field in ‘Goods-IN’

              3. UoM’s
            3. If decimal inventory is allowed, then Qty. Per UoM of Base. UoM should be greater than the MinPick Qty. Per UoM else MinPick Qty. Per UoM should be greater
            4. UoM of B.UoM and Alt.UoM cannot be same
            5. If UoM of B.UoM and MinPick UoM are same then Alt.UoM’s UoM should be different from the former
            6. If UoM of ‘Alt. UoM’s , MinPick are same’, then ‘Qty. Per UoM of Alt. UoM’s should be multiple of MinPick Qty. per UoM’(other than that qty)
            7. Once ‘Alt. UoM’s’ are configured, cannot edit ‘B.UoM, MinPick UoM’
            8. ‘B.UoM, MinPick UoM cannot be deleted’ until ‘Alt. UoM’s are deleted’
            9. UoM Types configured for an item cannot be modified/deleted, ‘once the Goods Movement(IN/OUT) transaction is done’
            10. When item code is configured in PO/SO then cannot edit UoM, quantity but can add a new item
          10. Business Rules:
            1. When an account user creates new item then all the tenants under that account should come in Tenant drop down
            2. When a tenant user creates new item then only tenant user mapped tenant should come in Tenant drop down
            3. Part number: Part number will be unique based on account and tenant level.
            4. Edit: Once material is mapped to PO, cannot modify the item details, supplier details, material storage parameter details and UoM details
      6. Kit Planner List
        1. Navigation: Home Master Data Kit Planner List
        2. Description
          1. This feature allows the user to view the list of kit items available and to request a new kit to be added in the kit planner before using the item in any of the system transactions. Kit Planner provides Search, Edit, Add/Edit Kit Items, Export to Excel functions based on the user role.
        3. Functionality
          1. The Items can be searched with a given Part Number.
          2. The ‘Add New Kit’ function allows to add parent kit (part no., supplier, description, BUoM/Qty.)
          3. The ‘Edit’ function provided in each of the item allows user to modify the kit details.
          4. The ‘Child Kit Items’ function in the Add/Edit Kit Items column allows to add child kit items to the parent (part no., kit UoM/Qty., Qty.)
            1. The ‘Edit’ function in each of the Kit items allows to modify the kit details. One or more child items with their basic details can be updated to the Parent.
          5. The entire kit planner list that is displayed can be exported as an ‘Excel file’ for reference.
        4. Business Rules
          1. Only the items with approved and active status, can be configured in kit planner and also will be available in the list.
          2. On modifying an already configured kit parent, system will remap the child items to the newly entered parent material.
          3. Item configured in kit planner cannot be child to the same kit but can be a child item for another kit planner item.
          4. Once kit planner is mapped to PO/SO, then cannot modify the item details, active/delete status.
      7. Location Manager
        1. Navigation: Home Master Data Location Manager
        2. Description
          1. Interactive location management and warehouse zone floor map tool. Enables the warehouse operators to view the complete zone floor map from the last storage unit (locations/bins) to the highest classification (Racks) in its hierarchy in an interactive map. Operators can create, modify, add and delete locations based on the business logic configured. Bulk Locations can also be created. Operators can search and plot locations based on the Item, Tenant.
          2. The following screen is used to manage the location(s) in a warehouse. User can get the location/Bin by selecting warehouse and Zone (Site code)
        3. Functionality
          1. The following screen displays the locations on selection of warehouse and Zone having Rack, Bay and Location/Bin. (Ex: S101V01).This interactive tool help the user to view the Item and their quantities in a bay by placing the mouse pointer over the bin.
          2. New Location: System creates new locations i.e., Bulk locations in a ‘Warehouse’ and its corresponding ‘Zone (Site Code)’ using slider selection of Rack, Bay and Location/Bin available as shown in the following figure.
          3. Location Code Genealogy: For example, consider the created location as ‘WM01A0102’ for which

            4. WM - Zone code

            01 – Rack

            A – Level

            01 – Column

            02 - Bin

          4. Delete: Allows deleting the empty Location(s)/Bin(s) in a Warehouse and its corresponding Zone (Site code)
          5. Modify
            1. Permits to view and modify the location(s) details in separate pop-up as shown above.
            2. ‘Allow Mixed Material’ is checked in Green, then Mixed materials are allowed to be placed in that location while goods receipt.
            3. ‘Allow Mixed Material’ is unchecked, then Fixed materials are allowed to be placed in that location while goods receipt.
            4. For a location ‘Is Quarantine’ is selected as ‘Yes’, then only the damaged/discrepancy materials are to be received in that location.
            5. ‘Fast- Moving Location’ is checked in Green, then Fast-moving goods will be placed in that location while Goods receipt ‘Active Location’ is checked in Green, then the location will be blocked and will not be shown on selection of warehouse and Zone Specific Tenant and Supplier under Tenant can be mapped to the selected location in Modify
            6. Once the Location/Bin details are updated, the system terminates the screen.
          6. Bulk Modify
            1. Permits to Bulk modify the location(s) details in separate pop-up as shown below.
            2. ‘Allow Mixed Material’ is checked in Green, then Mixed materials are allowed to be placed in that location while goods receipt.
            3. ‘Is Quarantine’ is checked in Green, then only the damaged/discrepancy materials are to be received in that location.
            4. ‘Fast- Moving Location’ is checked in Green, then Fast-moving goods will be placed in that location while Goods receipt
            5. ‘Active Location’ is checked in Green, then the location will be blocked and will not be shown on selection of warehouse and Zone
            6. Specific Tenant and Supplier under Tenant can be mapped to the selected location in Modify
            7. Bulk Modify is performed based on Rack wise – columns and levels under the rack and all the above mentioned updates can be applied based on User selection.
          7. Add
            1. Permits to Add New Locations by providing details in separate pop-up as shown below Permits to Add New Locations by providing details in separate pop-up as shown below
            2. Search: System shows ‘Location/Bin(s)’ on selection of search category in a ‘Warehouse’ and its corresponding ‘Zone (Site code)’ as shown in the following screens
            3. Select Search Category Tenant. Will fetch the Locations mapped to a specific tenant and will be highlighted in ‘Blue’ color
            4. If Select Search Category Part Number. Will fetch the Locations where the Part Number is present and will be highlighted in ‘Turquoise color
            5. Green color indicates Filled locations.
            6. Pale purple color indicates Empty locations.
            7. On moving the mouse pointer over the filled locations, it displays the Parts of a supplier present in the location and the current quantity under Available, In-OH, Ob-OH.
            8. On moving the mouse pointer over the filled locations, it displays the Bin Replenishment Data.
            9. On moving the mouse pointer over the filled locations, it displays the mapped Suppliers.
            10. Business Rules
              1. Location Code must start with zone code and based on genealogy
              2. Locations can be created by Warehouse Manager / Account Admin
              3. If a location is marked as Fast-Moving location, then the same will be suggested during Goods receipt if the item is configured with Bin replenishment details in Item master
              4. A location cannot be deleted when Items are currently present
              5. User type with ‘ View Only’ Role can only View the Location Manager screen

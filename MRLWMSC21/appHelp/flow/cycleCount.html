    1. Cycle Count
      1. start with the Cycle count, go to Orders Inventory Cycle Count List.
      2. The below screen is displayed.
      3. Click on “Add +” button.
      4. Select the mandatory Account, Warehouse, Cycle Count, Cycle Count Name, Frequency and Cycle Count Duration and Valid From.
      5. User can configure Cycle Counts for logged in account only.
      6. All the configured Cycle Counts for respective Account will display in Cycle count Configuration list.
      7. Cycle Count Name should be unique.
      8. All mapped WH’s to respective account should display in Warehouse dropdown. User can select a WH from the dropdown list.
      9. ‘Is Blind Cycle Count’ checkbox should be auto checked on Cycle Count details form. If Cycle Count is configured as Blind, ‘Available Qty.’ shouldn’t display on capture screen in CC transaction.
      10. User can uncheck ‘Is Blind Cycle Count’ checkbox depending on requirement. If Cycle Count is configured as non-blind, Available Qty. should display on capture screen in CC transaction.
      11. After configuring a new cycle count, below grid “Entity Configuration” should get enabled.
      12. Cycle Count Duration is more than Frequency configured for that CC error message “Cycle Count Duration Should Be Less Than Or Equal To Frequency.” Should come.
      13. An alert message is displayed as shown above.
      14. This data is displayed in the list as shown below.
      15. Now click on edit icon in the action column and below screen is displayed.
      16. Click on “Add +” button and the below shown popup is displayed.
      17. Give the mandatory fields and click on “Create” button. An alert message is generated as shown below.
      18. User can configure one or more entities for a cycle count.
      19. Entity configuration should be unique.
        Entity configuration should be unique.
        1. Entity configuration should be unique.
        2. If at least one transaction/cycle is initiated (in CC Transaction module) for a configured cycle count, then user can edit only the ‘Valid To’ date and ‘Frequency’ of the respective Cycle Count.
        3. If at least one transaction/cycle is completed, then user can edit only the ‘Valid To’ date.
        4. Cycle Count Duration Should Be Less Than or Equal To Frequency.
        5. 5) Validation at the time of changing Valid To: Valid to should not be less than the maximum Planned End Date of any cycle counts in the series as per the formula:

          PED = PSD of this Series + (Duration -1) Days

        6. Once all transactions for cycle count is marked completed, then no more changes are permitted for respective CC.
        7. User can configure one or more entities for a cycle count.
          Below are the cases:
          1. Case1: If user configures ‘Zone’ as an entity, on capture screen locations belong to that zone under WH should come only. And materials belongs to account should come.
          2. Case2: If user configures ‘Material’ as an entity, on capture screen locations belong to WH should come and configured material should come.
          3. Case3: If user configures ‘Tenant’ as an entity, on capture screen locations belong to WH should come and material should come based on Tenant.
          4. Case4: If user configures ‘Supplier’ as an entity, on capture screen locations belong to WH should come and materials should come based on Supplier.
        8. Now click on Cycle Count Capture. The below list is displayed.
        9. Now click on “Initiate” button.
        10. Below shown alert message is displayed.
        11. Now in the action column two buttons are displayed as shown below.
        12. Now click on “Capture” button, and the below shown popup is displayed.
        13. Give the mandatory (Location, material and Qty.) fields and save the data.
        14. The confirmation popup is displayed confirming the release of dock as shown below.
        15. Once confirmed the “Capture” button changes to “Captured” in the action column. Screen shown below.
        16. Now click on “Complete” button to finish the Cycle Count process.
        17. Now the completed cycle count’s planned start and end dates are shown in the above displayed screen.
          Business Rules for CC Capture:
          1. Calculations of PSD and PED should be computed as per the below formulas:
            2. For First CC in Series:
            PSD = Valid From of Config Table
            PED = PSD of Series + (Duration -1) Days

            For Nth CC in series:
            PSD = AED of (N-1)th Series + (Frequency) Days
            PED = PSD of this Series + (Duration -1) Days
          2. CC Transaction List:
            1. Once a Cycle Count is configured for an Account(logged In), it will display on Cycle Count Transaction page.
            2. Cycle Counts of the logged in account should display only.
            3. Beside Search, a "Show All" Checkbox is displayed, to show completed CC also in CC transaction list.
            4. Display order is descending Order for list elements when “Show All” checkbox is not checked and here only CC’s with status Planned and initiated should display.
            5. Once an entity is configured to that cycle count, then only user will be able to initiate it, else initiate button will not come on CC transaction page.
            6. If ‘Valid From’ is future date and entity is configured for that CC, initiate button won’t come before ‘Valid From’ date for that CC on CC transaction page.
            7. By clicking on initiate button, user should be able to initiate cycle count.
            8. After initiating cycle count, status should be changed to ‘Initiated’ from Planned.
          3. Edit CC :
            1. User can edit a cycle count, if status for CC is ‘Planned’. Once user has initiated the CC, user shouldn’t be able to edit CC.
            2. PED should be an auto computed label.
          4. Cycle Count Capture:
            1. Case1: If user configures ‘Zone’ as an entity, on capture screen locations belong to that zone under WH should come only. And materials belongs to account should come.
            2. Case2: If user configures ‘Material’ as an entity, on capture screen locations belong to WH should come and configured material should come.
            3. Case3: If user configures ‘Tenant’ as an entity, on capture screen locations belong to WH should come and material should come based on Tenant.
            4. Case4: If user configures ‘Supplier’ as an entity, on capture screen locations belong to WH should come and materials should come based on Supplier.
          5. If no Zone is configured then all locations under configured WH should display.
          6. Location, Material, quantity are mandatory on Capture screen.
          7. Once capturing of data is done and user saves the entered data, button should changed to ‘Captured’ from ‘capture’.
          8. After capturing data, when user clicks on Complete button, it displays message that “All locations released and transaction completed successfully” and status should be ‘Completed’ and View icon start coming for that transaction. User can view transaction by clicking on that view icon.
          9. Beside Search, a "Show All" Checkbox is displayed, to show completed CC also in CC transaction list.
          10. If user checks "Show All" Checkbox, all transactions(Planned, Initiated and Completed) will display in ascending Order on CC transaction list.
          11. If “Show All” checkbox is unchecked, CC transaction list will display only CC’s with status Planned and initiated should display in descending order.
          12. Once an entity is configured to that cycle count, then only user will be able to initiate it, else initiate button will not come on CC transaction page.
          13. User shouldn’t be able to initiate cycle count before the Planned Start date.
          14. User should be able to edit PSD to any date between the CC Configuration PSD & PED.
          15. If Cycle Count is configured as Blind, then Available Quantity shouldn’t display, for non-blind cycle counts Available Quantity should display on capture screen.
          16. Once user clicks on save button, application will ask for releasing locations, if user clicks on Yes button, data entered should be saved and locations should be released successfully.
          17. If user will add same material on same location in a single transaction, it will be display in one single record.
          18. Once user clicks on save button, application will ask for releasing locations, if user clicks on No button, data entered should be saved but locations shouldn’t be released.
          19. After completing one transaction, status for transaction should be ‘Completed’ and entry for next transaction/sequence should come in transaction list.
          20. Once all transactions are completed for respective CC(as per Valid From and Valid To date in CC configuration), entry for next transaction/sequence should not come in transaction list.
          Hand Held Process for CC Transaction
          1. Capturing is of two types in HHD -
            1. Manual mode – Capturing quantity is user entry
            2. Auto mode – Capturing quantity is always 1 when item is scanned
          2. Capturing can happen with respect to Location.
          3. Scanned containers and locations must be under the defined warehouse of the User
          4. Scanned items must belong to respective account based upon entities configured.
          5. Location, container, item and quantity are mandatory while capturing data through HHD.
          Cycle Count Report
          1. Once a Cycle Count is marked as Completed, user can see CC report for that sequence/transaction by clicking on view icon in CC Transaction page for that sequence.
          2. User can navigate to REPORTS Audit Cycle Count Report
          3. Here user can see CC report for all the CC conducted till date.
          4. By default data for all CC’s will display.
          5. User can also filter CC, as per their requirements.
          6. On CC Report page, three filters are displayed:-

              A. Cycle Count

              B. Cycle Count Name

              C. Sequence No.

            7. User can select filters - Cycle Count, Cycle count Name and sequence no and filter cc report data.
